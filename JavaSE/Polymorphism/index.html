Java中的多态

发表于 2015-09-24   |   分类于 JavaSE   |  

多态：可以理解为事物存在的多种形态。

多态的体现

父类的引用指向了自己子类的对象
父类的引用也可以接受自己子类的对象

多态的前提

必须类与类之间有关系，要么继承，要么接口
通常还有一个前提，就是存在覆盖

多态的好处

大大提高了程序的扩展性

多态的弊端

提高了扩展型，但是只能使用父类的引用访问父类的成员。

多态的应用

Animal a = new Cat(); //类型提升向上转型

如果要调用猫的特有方法，需要强制将父类的引用转成子类类型。
Cat c = (Cat) a;

在多态中成员函数的特点

在编译时期：参阅引用型变量所属的类中是否有调用的方法，如果有，编译通过。
在运行时期，参照对象所属的类中是否有调用的方法。

Objext类

每个类都使用作为超类。

equals(Object obj)

判断对象是否相等，如果自定义类中也有比较相同的功能，没有必要重新定义，只要沿袭父类中的功能，建立自己特有的比较内容就可以，这就是覆盖。
